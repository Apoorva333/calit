# Defensive Security Audit — calit

- **Date:** 2026-06-12
- **Service:** calit (Quarkus / Java 25, Maven), version 1.2.0
- **Auditor:** java-security-audit skill (defensive threat review)
- **Mode:** Read-only. No exploitation, no code changed.
- **Method:** OWASP Top 10 lens applied to every entry point across 6 focus areas (input validation, authz, SSRF, deserialization, secrets/logging, dependencies).

---

## 1. Security context (system map)

**Stack:** Quarkus 3.36.2, Hibernate ORM Panache + PostgreSQL, Flyway-owned schema, Qute templates, Quarkus Mailer, Quarkus Scheduler, Google Calendar API client, BouncyCastle (argon2id). Multi-tenant ("owner"-scoped) calendar-booking app.

**Entry points:**
- Public (no auth): booking flow (`PublicResource` HTML + `BookingResource` `/api/bookings` JSON), meeting-type pages, `/login`, `/setup` (first-run only), `/signup` (flag-gated), `/api/google/callback`, health.
- Authenticated `user` role (`/me`, `/me/*`, `/api/google`, `/api/google/*`): owner self-service — meeting types, availability, overrides, bookings approve/decline, settings, Google connect.
- Admin role (always user id 1): `/me/users` user management.
- Background: reminder + pending-expiry schedulers.

**Data stores:** PostgreSQL (bookings + invitee PII, app users + argon2id hashes, Google OAuth tokens, meeting config).

**Sensitive assets:** invitee PII (name/email/notes), user password hashes, **Google OAuth refresh/access tokens**, session encryption key, OAuth state HMAC secret, SMTP creds, Turnstile secret.

**Trust boundaries:** anonymous internet → public booking; authenticated owner → own tenant data; admin → user management; app → Google/Cloudflare/SMTP (egress).

**Overall posture:** Strong. Owner-scoping (tenant isolation) is applied uniformly with no cross-owner IDOR found. Parameterized Panache queries everywhere (no SQL/HQL injection). Qute auto-escaping intact (no XSS). OAuth state HMAC-signed + constant-time verified. argon2id with correct OWASP params. The real risk cluster is **secret/token hygiene + prod misconfiguration**, plus missing input bounds and absent SCA scanning.

## 3. Findings (detail)

### SEC-SECRET-02 — Google OAuth refresh/access tokens stored plaintext at rest
- **Severity:** High · **OWASP:** A02 Cryptographic Failures
- **Affected:** `src/main/java/com/calit/google/GoogleCredential.java:27-37`; migrations `V3__google.sql`, `V9__google_multi_account.sql` (`refresh_token TEXT NOT NULL`, `access_token TEXT`)
- **Description:** Long-lived offline refresh tokens and access tokens persisted as plain `TEXT`, no app-level encryption. `GoogleTokenService` writes raw token strings directly.
- **Impact:** Any DB read access (backup theft, compromised DB creds, the global SQL logging in SEC-SECRET-01) yields long-lived Google Calendar refresh tokens = full calendar read/write per owner, persistent until revoked.
- **Recommendation:** Encrypt tokens at rest — envelope encryption with a KMS/app key, or pgcrypto. At minimum enforce DB-at-rest encryption + restricted DB roles and exclude tokens from SQL/param logging. **Flyway rule: new `V*` migration, never edit applied ones.**
- **Test:** Persist a credential, read the row via raw SQL, assert ciphertext (not the token) is stored.

### SEC-SECRET-03 — Insecure dev-default OAuth state-secret can silently boot prod
- **Severity:** High · **OWASP:** A05 Misconfiguration / A02
- **Affected:** `src/main/resources/application.properties:28` (and session key 85-86)
- **Description:** `google.oauth.state-secret=${GOOGLE_OAUTH_STATE_SECRET:dev-only-insecure-state-secret-change-me}` is a **global** line with an insecure default and **no `%prod` override**. The session encryption key *does* have a `%prod` no-default override (fails fast), but the state-secret does not — an unset `GOOGLE_OAUTH_STATE_SECRET` in prod falls back to the publicly-known dev key. No startup validation rejects dev-default secrets.
- **Impact:** With the known HMAC key, an attacker forges OAuth `state` and binds a victim's callback to an arbitrary `ownerId` → account-takeover of the Google-connect flow.
- **Recommendation:** Add `%prod.google.oauth.state-secret=${GOOGLE_OAUTH_STATE_SECRET}` (no default). Add a `%prod` `@Observes StartupEvent` validator that fails boot if any security secret equals its `dev-only-insecure-*` default or is too short.
- **Test:** Boot the prod profile with empty `GOOGLE_OAUTH_STATE_SECRET`; assert boot fails.

### SEC-AUTHZ-01 — Admin self-lockout / last-admin removal with no in-app recovery
- **Severity:** High · **OWASP:** A01 Broken Access Control (privilege-management invariant)
- **Affected:** `src/main/java/com/calit/web/UsersResource.java:84-118` (`grantAdmin`/`revokeAdmin`/`lock`/`unlock`)
- **Description:** Endpoints are correctly `@RolesAllowed("admin")`, but no invariant stops an admin from revoking their own admin role, locking their own account, or removing the last enabled admin. `EnabledUserAugmentor` downgrades a disabled user to anonymous on the next request; `SetupResource` 404s once any user exists → **no recovery path short of direct DB access.**
- **Impact:** Single click can irreversibly destroy all admin capability. Not externally exploitable (needs an admin), but severe and easy to trigger accidentally.
- **Recommendation:** Reject `revoke-admin`/`lock` when target == current principal or target is the last enabled admin (`AppUser.count("isAdmin = true and enabled = true") <= 1`); re-render with an error.
- **Test:** Sole admin POSTs `/me/users/1/revoke-admin` and `/lock` → rejected, state unchanged, `/me/users` still reachable.

### SEC-DEP-01 — No SCA / dependency + image vulnerability scanning in CI
- **Severity:** High · **OWASP:** A06 Vulnerable & Outdated Components
- **Affected:** `.github/workflows/ci.yml` (no scan job); no Dependabot/CodeQL config
- **Description:** CI builds, tests, and publishes multi-arch images with zero vuln scanning. Renovate raises update PRs but does not gate on CVEs.
- **Impact:** Vulnerable Java/npm/base-image components can ship to GHCR undetected.
- **Recommendation:** Add a blocking Trivy fs+image scan (fail on CRITICAL/HIGH); enable Dependabot alerts + CodeQL; add `"vulnerabilityAlerts": {"enabled": true}` to renovate.json.

### SEC-SECRET-01 — SQL logging enabled globally (PII/SQL in prod logs)
- **Severity:** Medium · **OWASP:** A09 Logging Failures
- **Affected:** `src/main/resources/application.properties:5` — `quarkus.hibernate-orm.log.sql=true` (unprofiled → applies to prod)
- **Impact:** Every query (invitee PII, usernames, token-table reads) emitted to prod logs; recon aid + GDPR exposure in log aggregation.
- **Recommendation:** Scope to `%dev.` (and `%test.` if needed); never enable bind-parameter logging in prod.

### SEC-SECRET-04 — No CSRF tokens on authenticated state-changing POSTs
- **Severity:** Medium · **OWASP:** A05 / CSRF
- **Affected:** `pom.xml` (no `quarkus-rest-csrf`); all `@POST` forms in `web/*`, `user/SetupResource`, `google/GoogleCalendarResource`
- **Description:** Only `SameSite=Lax` protects privileged POSTs. No CSRF-token mechanism. Lax is insufficient against same-site/subdomain attackers and breaks entirely if cookie is ever set to `SameSite=None`.
- **Recommendation:** Add `quarkus-rest-csrf`, emit/verify per-form tokens on all authenticated POSTs; keep Lax as defense-in-depth.

### SEC-INPUT-01 — Unvalidated invitee email enables header/ICS injection
- **Severity:** Medium · **OWASP:** A03 Injection
- **Affected:** `PublicResource.java:166-199`, `BookingService.java:141-178`, `EmailService.java:271-278`, `IcsBuilder.java:44`
- **Description:** `inviteeEmail` accepted with no `@Email`, no length cap, no CRLF check; persisted then passed verbatim to `Mail.withHtml(to,…)` and the ICS `ORGANIZER:mailto:` line. `IcsBuilder.escape()` covers SUMMARY/LOCATION but not UID/ORGANIZER. No bean validation anywhere in `src/main/java` (`grep @Valid/@Email/@Pattern` → none).
- **Impact:** Crafted address may inject recipients/headers (BCC exfiltration / relay) or smuggle CRLF into the ICS, depending on SMTP backend parsing.
- **Recommendation:** Validate single-address + length ≤254 + reject CR/LF/commas before persist (`@Email @Size(max=254) @Pattern(regexp="[^\r\n]+")` + `@Valid`). CRLF-escape every ICS-interpolated value.
- **Test:** POST `inviteeEmail = "a@b.com\r\nBcc: attacker@evil.com"` → 400, no extra recipient.

### SEC-DEP-02 — Container/service images pinned by floating tag, not digest
- **Severity:** Medium · **OWASP:** A06
- **Affected:** `Dockerfile:4,16,37` (`oven/bun:1`, `liberica…:jdk-26-musl`, `…:jre-26-musl`); `docker-compose.yml:9` (`postgres:18`)
- **Recommendation:** Pin to `@sha256:` digests; let Renovate manage digest bumps.

### SEC-DEP-05 — Runtime container likely runs as root
- **Severity:** Medium · **OWASP:** A06 / A05 hardening
- **Affected:** `Dockerfile:37-50` (no `USER` directive)
- **Recommendation:** Add non-root `USER 1001` + correct `/app` ownership (verify Liberica base default user first).

### SEC-DEP-04 — Hand-pinned Java deps need periodic CVE verification
- **Severity:** Medium (verify) · **OWASP:** A06
- **Affected:** `pom.xml:67-91` — google-api-client 2.9.0, google-oauth-client 1.39.0, google-api-services-calendar v3-rev20260517-2.0.0, **bcprov-jdk18on 1.84**, quarkus-bom 3.36.2
- **Note:** No fabricated CVEs; verify each against OSV.dev/NVD (BouncyCastle has CVE history). Covered operationally by SEC-DEP-01 scan gate.

### SEC-AUTHZ-02 — Public `/api/bookings` parallels the web booking flow (regression-drift risk)
- **Severity:** Medium · **OWASP:** A01
- **Affected:** `BookingResource.java:29-64`
- **Description:** No present cross-tenant hole (owner resolved from body `{user}`, type looked up within that owner; abuse guards live in `BookingService.book`). Risk is duplication drift: a control added only to the web flow won't apply here. `reschedule`/`cancel` are authorized solely by the `manageToken` (CSPRNG UUID — OK) and do not re-check abuse guards.
- **Recommendation:** Document/test that both entry points funnel through identical `BookingService` guards; treat `manageToken` as a bearer credential.

### SEC-SECRET-05 — No audit logging of privileged admin actions
- **Severity:** Low–Medium · **OWASP:** A09
- **Affected:** `web/UsersResource`, `web/AdminResource`, `user/SetupResource`, `google/GoogleOAuthResource`; only 2 log statements exist app-wide (`GoogleCalendarPort.java:96,151`). Failed logins are silent.
- **Recommendation:** Emit structured audit events (actor/action/target/ts/source-IP) for auth outcomes + all privileged mutations; never log secrets.

### SEC-INPUT-02 — No length/type bounds on free-text booking inputs
- **Severity:** Low · **OWASP:** A03 (improper input validation)
- **Affected:** `Booking.java:33,72-74`, `PublicResource.java:184-191`, `BookingService.java:253-263`
- **Description:** `inviteeName` + custom-field answers unbounded; declared field types (NUMBER/EMAIL/PHONE) not enforced server-side; answer keys not allowlisted against the type's fields.
- **Recommendation:** Server-side length caps (name ≤200, answer ≤2000, total payload bound) + per-field-type format validation + reject unknown answer keys.

### SEC-SSRF-01 — Outbound HTTP calls lack connect/read timeouts
- **Severity:** Low · **OWASP:** A10 (availability hardening; not exploitable SSRF — destinations fixed)
- **Affected:** `TurnstileVerifier.java:36`, `GoogleTokenService.java:216`, `GoogleCalendarClientFactory.java:34-35`
- **Description:** Clients built with library defaults, no timeouts/size caps/redirect policy. Turnstile call is on the synchronous booking path.
- **Recommendation:** Set `connectTimeout` + request `timeout`; bound redirects (`Redirect.NORMAL`) and Turnstile response size.

### SEC-INPUT-03 — `error` query param reflected in OAuth callback
- **Severity:** Low · **OWASP:** A03
- **Affected:** `GoogleOAuthResource.java:45-48` — reflects raw `?error=` into a `text/plain` 400. Not HTML-executable under `text/plain`; content-spoofing only.
- **Recommendation:** Log server-side, return fixed message, add `X-Content-Type-Options: nosniff`.

### SEC-AUTHZ-03 — `MeOwnerFilter` returns instead of failing closed on missing user
- **Severity:** Low · **OWASP:** A01 (defense-in-depth)
- **Affected:** `MeOwnerFilter.java:39-43` — when an authenticated principal has no `AppUser` row, the filter returns without setting `CurrentOwner` and without aborting; relies on downstream null-handling (effectively 401 / empty results). Upstream augmentor makes this near-unreachable.
- **Recommendation:** Abort 401 here (fail closed).

### SEC-DEP-03 — npm build deps use caret ranges
- **Severity:** Low · **OWASP:** A06
- **Affected:** `package.json:9-11` — mitigated by committed `bun.lock` + `--frozen-lockfile` in Dockerfile & CI; no JS shipped at runtime. Keep frozen-lockfile enforcement.

## 4. Verified OK (controls confirmed correct)

- **Tenant isolation / IDOR — SAFE.** Every `/me/*` mutator routes through `requireType` / `requireOwnedBooking` / inline owner check before mutating; cross-owner read/edit/delete by id or slug is blocked. Public approve/decline not exposed.
- **No SQL/HQL injection.** All Panache `find/list/count/delete` use positional params (`?1`); no string-built queries; no raw `EntityManager` user-query building.
- **No XSS.** Qute auto-escaping relied on consistently; the only `.raw` outputs bind static `Layout` constants, never user data; no triple-brace/`|raw`/`|safe` on user input.
- **OAuth state — SAFE.** HMAC-SHA256 signed, `MessageDigest.isEqual` constant-time verify, 10-min TTL, owner recovered from signed state not session (weakness = the default key, SEC-SECRET-03).
- **OAuth redirect_uri / callback — SAFE.** `redirect_uri` server-config only; callback redirects to fixed `/me/google`; Google + Turnstile base URLs are fixed constants — no SSRF / open-redirect.
- **No unsafe deserialization.** No `ObjectInputStream`/`readObject`; no Jackson default typing / `@JsonTypeInfo`; sole JSONB column is a non-polymorphic `Map<String,String>`; Google token JSON parsed strictly from TLS source.
- **Password storage — argon2id** (m=19456 KiB, t=2, p=1, 16-byte SecureRandom salt, 32-byte hash, constant-time compare, MCF-encoded).
- **Secrets in repo — none.** `.env` gitignored + untracked; only placeholder `.env.example`; `application.properties` all `${ENV}`-driven; DB password no-default (fail-fast); compose `${DB_PASSWORD:?…}`, Postgres not host-published.
- **Exception mappers** return clean domain messages only — no stack traces/SQL/internal ids.
- **Setup/signup gating** correct: `/setup` 404s once any user exists (both verbs); `/signup` honors `SIGNUP_ENABLED` on GET+POST, signup users non-admin.
- **Invitee manage links** use CSPRNG `UUID.randomUUID()` tokens, not sequential ids.
- **No mass assignment** — entities constructed manually, owner/id/status set server-side; booking API uses explicit `BookRequest` record.
- **Dependency provenance** — quarkus-bom manages transitive versions, no custom `<repositories>`, multi-stage Docker with no secrets in layers, BouncyCastle confirmed used (argon2id).

3. **SEC-SECRET-02** — encrypt Google tokens at rest (new Flyway `V*`).
4. **SEC-AUTHZ-01** — last-admin / self-lockout guard.
5. **SEC-DEP-01** — add Trivy + Dependabot/CodeQL scan gate to CI.
6. **SEC-SECRET-04 / SEC-INPUT-01** — CSRF tokens; validate invitee email + bound inputs.
7. Hardening: SEC-DEP-02/05, SEC-SSRF-01, SEC-SECRET-05, remaining Lows.
