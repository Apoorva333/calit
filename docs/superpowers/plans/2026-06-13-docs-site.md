# calit Documentation Site Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a public Astro Starlight documentation site for calit, hosted on GitHub Pages at `https://asm0dey.github.io/calit/`, reusing the existing marketing landing as its homepage.

**Architecture:** Self-contained Astro Starlight project in a `docs-site/` subdirectory (own Bun deps, isolated from the root Quarkus app). A custom `src/pages/index.astro` ports the existing Qute landing (auth stripped, nav re-pointed). Starlight serves themed doc pages. A GitHub Actions workflow builds and deploys to Pages on every push to the `docs-site` branch.

**Tech Stack:** Astro + Starlight, Bun, GitHub Actions (`withastro/action` + `actions/deploy-pages`), Playwright (screenshot capture).

**Working branch:** `docs-site` (already created and checked out). The name `docs` is unavailable — it collides with the existing `docs/compose-example` ref.

**Reference spec:** `docs/superpowers/specs/2026-06-13-docs-site-design.md`

## Task 5: GitHub Actions deploy workflow

**Files:**
- Create: `.github/workflows/docs.yml`

- [ ] **Step 1: Write the workflow**

Create `.github/workflows/docs.yml`:

```yaml
name: Deploy docs site

on:
  push:
    branches: [docs-site]
    paths:
      - 'docs-site/**'
      - '.github/workflows/docs.yml'
  workflow_dispatch:

permissions:
  contents: read
  pages: write
  id-token: write

# Allow one concurrent deployment; cancel in-progress runs for the same ref.
concurrency:
  group: docs-pages
  cancel-in-progress: true

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Build with Astro
        uses: withastro/action@v3
        with:
          path: ./docs-site
          package-manager: bun@latest

  deploy:
    needs: build
    runs-on: ubuntu-latest
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
```

`withastro/action@v3` runs `bun install` + `astro build` in `./docs-site` and uploads the Pages artifact automatically; the deploy job publishes it.

Body must cover (these are correctness-critical for production):

- calit listens on **plain HTTP on port 8080** inside the container; TLS is terminated at the proxy.
- **Secure-cookie requirement:** the login cookie's `Secure` flag is derived from `request.isSSL()`. Behind a TLS-terminating proxy that is false unless the app trusts the proxy's forwarded headers. calit enables `quarkus.http.proxy.proxy-address-forwarding=true` in the `%prod` profile so `X-Forwarded-Proto` is honored and the login cookie is correctly marked `Secure`. Therefore the proxy **must** set `X-Forwarded-Proto` (and `X-Forwarded-For`/`Host`).
- **Hardening caveat:** trusting forwarded headers is only safe when calit is reachable **only** via the proxy. If the container port is otherwise reachable, restrict trust with `quarkus.http.proxy.trusted-proxies=<proxy CIDR>`.
- Set `APP_BASE_URL` to the public HTTPS URL (used for manage links + Google redirect URIs).
- No WebSocket configuration is needed — calit ships no runtime JavaScript.
